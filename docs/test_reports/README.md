# Historical test and migration reports

The documents in this directory are retained as implementation history. They describe the repository at the time each dated change was made and are **not authoritative documentation for the current codebase**.

Paths, compatibility fields, fallback mechanisms and recommended follow-up work recorded in an older report may have been changed or removed by later pull requests.

Use the following maintained documents for the current project contract:

- `README.md` — current project baseline and verification entrypoints;
- `docs/architecture.md` — current subsystem ownership and data flow;
- `docs/car_catalog.md` — current car model, variant and specification contracts;
- `docs/vehicle_model.md` — current vehicle simulation model;
- `docs/continuous_integration.md` — current CI behavior;
- `docs/windows_export.md` — current export behavior;
- `docs/roadmap.md` — current completed and deferred work.

When a historical report conflicts with maintained documentation or production code, the maintained documentation and validated production contract take precedence.
